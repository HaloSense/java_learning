# Day 6

## 方法

方法是程序当中最小的执行单元。

把重复的代码写在方法当中，提高复用性、可维护性。

### 方法定义

把一些代码打包在一起，叫做方法定义。

### 方法调用

通过调用方法，可以运行方法里打包好的代码。

### 带参方法

可以通过设置参数的方式向方法内传递数据。

调用方法时，方法的参数类型和数量必须与定义一致，否则会报错。

### 形参与实参

形参：全称形式参数，指**方法定义**中的参数

实参：全程实际参数，指**方法调用**中的参数

方法调用时，形参和实参的类型和数量必须一致，否则会报错。

### 带有返回值的方法

返回值可以将方法内某些变量传递回方法调用的位置，结合参数传递可以实现方法内外的数据交互。

`return 变量名`

 ### 有返回值方法的调用格式
 
1. 直接调用
   ```java
   public class MethodDemo {
        static void main() {
            sumInt(3, 5);
        }
    
        public static int sumInt(int num1, int num2){
            return num1 + num2;
        }
    }   
   ```

2. 赋值调用
   ```java
   public class MethodDemo {
       static void main() {
           int sum = sumInt(3, 5);
       }
   
       public static int sumInt(int num1, int num2){
           return num1 + num2;
    }
   }   
   ```
  
3. 输出调用
   ```java
   public class MethodDemo {
        static void main() {
            System.out.println(sumInt(3, 5));
        }
   
        public static int sumInt(int num1, int num2){
            return num1 + num2;
        }
    }   
   ```

### 方法的注意事项

1. 方法不调用就不执行
2. 方法和方法之间是平级的关系，不能嵌套
3. 方法的编写顺序和执行顺序无关
4. 如果方法返回类型为`void`，表示该方法没有返回值，可以省略`return`语句。
   如果需要写`return`，那么后面不能跟着具体数据。此时`return`会直接结束方法。
5. `return`下面不能写代码，因为永远执行不到。

### 方法的重载

在同一个类中定义多个相同名称的方法，这些方法的功能相同。每种方法具有不同的参数类型或参数个数。
这些方法构成重载关系。

关键点：**同一个类**中，若干个方法**名称相同**，且方法**参数类型或参数个数不同**。

方法是否构成重载关系**与返回值无关**。

## Extra: 方法的内存图

### 方法在运行时的内存调用

方法主要在栈中运行。

栈 -> 先进后出，后进先出

main方法首先入栈，然后调用的其他方法依次入栈，执行完毕后即出栈，main方法最后出栈。

### 方法在传递数据时的内存调用

#### 基本数据类型 VS 引用数据类型

基本数据类型中存储的是**真实的数据**。基本数据类型赋值给其他变量时，传递的是真实值。

引用数据类型中存储的是**指向其他空间的地址值**，真实值存储在其他空间中。
引用数据类型赋值给其他变量时，传递的是地址值。

传递基本数据类型时，传递的是真实值，形参的改变**不会影响**实参。

传递引用数据类型时，传递的是地址值，形参的改变**会影响**实参。
